# DGX H100 SuperPOD Thermal Surrogate

A data-driven surrogate model that replaces a full CFD solve of a DGX H100
SuperPOD data center with a fast neural network forward pass (~ms vs ~hours).

Given 10 operating parameters (inlet velocity, supply temperature, and 8
chassis load fractions), the model predicts the midplane temperature and
velocity fields over the entire 2-rack slice.

---

## Architecture

```
10 scalars  →  MLP Encoder  →  latent seed (128×4×7)
                                      ↓
                             CNN Decoder (5× ConvTranspose2d + ResBlocks)
                                      ↓
                   bilinear upsample to 111×194
                         ↙            ↘
             T field (1×111×194)   U field (2×111×194)
             Temperature [K]       Velocity Ux, Uy [m/s]
```

The encoder is intentionally shallow (10 inputs → 3 dense layers) and the
decoder carries the representational capacity, matching where the spatial
reconstruction challenge actually lives.

## Dataset

| File | Cases | Role |
|------|-------|------|
| `dataset.npz` | 500 | Training set |
| `test_dataset.npz` | 50 | Validation / evaluation |

Both files share the same array layout:

| Array | Shape | Description |
|-------|-------|-------------|
| `params` | (N, 10) | Input parameters (physical units, unnormalised) |
| `T_fields` | (N, 1, 111, 194) | Midplane temperature [K] |
| `U_fields` | (N, 2, 111, 194) | Midplane velocity [m/s], channels = (Ux, Uy) |
| `grid_x` | (194,) | x-coordinates of the midplane grid [m] |
| `grid_y` | (111,) | y-coordinates of the midplane grid [m] |
| `case_ids` | (N,) | CFD case identifiers |
| `splits` | (N,) | Split label (e.g. `"train"`, `"test"`) |

### Input parameters

| Index | Name | Range | Units |
|-------|------|-------|-------|
| 0 | `v_in_mps` | 1.0 – 4.0 | m/s |
| 1 | `T_sup_K` | 291 – 300 | K |
| 2–9 | `f1`–`f8` | 0.25 – 1.0 | — (chassis load fraction) |

The 500 training cases were generated by a Latin Hypercube Sample over this
parameter space and solved with `buoyantSimpleFoam` (k-ω SST, OpenFOAM v2406).

> **Note on dataset size:** `dataset.npz` (~500 cases × 111×194 fields) is
> several hundred MB and is tracked with [Git LFS](https://git-lfs.github.com/).
> `test_dataset.npz` (50 cases) is included directly.

## Usage

### Train

```bash
python surrogate_model_main.py --mode train
```

Trains on `dataset.npz` (500 cases), validates on `test_dataset.npz` (50
cases). Saves the best checkpoint to `checkpoints/best_model.pth` and loss
curves to `results/loss_curves.png`.

### Evaluate a saved checkpoint

```bash
python surrogate_model_main.py --mode eval
```

Loads `checkpoints/best_model.pth` and reports MAE, RMSE, and max-error
metrics on the 50 held-out cases. Also saves best/median/worst prediction
comparison plots to `results/`.

### Quick smoke test

```bash
python surrogate_model_main.py --mode train --epochs 10
```

### Custom data paths

```bash
python surrogate_model_main.py --mode train \
    --data /path/to/dataset.npz \
    --val_data /path/to/test_dataset.npz \
    --epochs 300
```

Each comparison plot shows three rows (T, Ux, Uy) × three columns (CFD
ground truth | Surrogate | Absolute error).

## Background

The CFD dataset was generated from a 2-rack DGX H100 SuperPOD slice with
hot-aisle containment modelled in OpenFOAM v2406 (`buoyantSimpleFoam`,
k-ω SST turbulence closure). The geometry includes 8 GPU chassis per rack
with individually variable load fractions. Full details are in the companion
paper (in preparation, *Applied Thermal Engineering*).
